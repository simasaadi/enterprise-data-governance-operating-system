# Change Notice Template

## Purpose
This template is used to communicate material data, reporting, metadata, access, or control changes to affected stakeholders.

## Summary
Provide a short, plain-language summary of the change.

## What is changing
Describe:
- the asset, report, rule, workflow, or access pattern changing
- what the new state will be
- whether this replaces or supplements the current state

## Why this is changing
Examples:
- control weakness identified
- approved business rule updated
- reporting consistency improved
- access risk reduced
- process simplification
- remediation from root cause review

## Who is affected
Examples:
- analysts
- managers
- stewards
- report consumers
- technical admins
- external recipients where applicable

## What users need to do
Examples:
- begin using a new report field
- stop using an old definition
- revalidate a downstream output
- request new access
- review updated guidance

## Risks or transition notes
Examples:
- short-term comparability issue
- historical values may differ after restatement
- temporary dual-run period
- manual review required until automation is updated
