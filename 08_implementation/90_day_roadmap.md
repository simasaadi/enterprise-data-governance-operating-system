# 90-Day Governance Roadmap

## Purpose
This roadmap defines a practical first 90 days for standing up the governance operating system in a controlled, usable, and adoption-focused way.

## Objective
The objective is to prioritize early actions that create operating clarity, visible control improvement, and reusable governance discipline without overwhelming teams.

## Roadmap principles
- start with highest-risk and highest-value assets
- establish role clarity before expanding documentation volume
- prioritize workflows and controls that change behavior
- use lightweight but structured artifacts early
- build visible wins in quality, access, and issue handling
- treat governance rollout as an operational change effort

## Day 1 to 30: establish the operating core

### Priorities
- confirm governance scope and sponsorship
- identify critical domains, reports, and assets
- confirm data owner and steward assignments
- launch core governance structure and escalation pathway
- stand up initial issue intake and access decision logging
- define initial high-priority data quality controls
- begin glossary and metadata capture for critical assets

### Key deliverables
- governance charter and structure confirmed
- decision rights model confirmed
- owner and steward assignments for priority assets
- issue intake log active
- access decision log active
- first-pass DQ control list and thresholds defined
- initial glossary and dictionary entries created

### Success indicators
- named accountable roles exist for priority assets
- issue and access decisions are no longer handled only through email
- first controls and thresholds are visible
- initial metadata foundation is in place

## Day 31 to 60: operationalize the core workflows

### Priorities
- run stewardship cadence and governance review forums
- execute recurring quality controls and capture exceptions
- use the issue workflow on real cases
- apply change impact assessment to material changes
- apply minimum-necessary access logic to new requests
- expand metadata and glossary coverage for shared assets
- begin KPI tracking and management reporting

### Key deliverables
- stewardship forum cadence running
- active issue triage and remediation tracking
- first access reviews and exception decisions documented
- change impact assessment in use
- first monthly KPI pack assembled
- metadata and glossary coverage increased for critical assets

### Success indicators
- governance artifacts are being used in practice
- recurring controls are producing evidence
- exceptions are visible and traceable
- access and change decisions are more disciplined
- KPI reporting begins informing management follow-up

## Day 61 to 90: stabilize, measure, and expand

### Priorities
- review repeat issues and root causes
- refine thresholds and control design based on evidence
- assess maturity baseline across dimensions
- confirm implementation gaps and next-wave priorities
- communicate lessons learned and adoption expectations
- prepare executive summary of progress, risk, and next steps

### Key deliverables
- root cause reviews completed for material recurring issues
- initial maturity assessment completed
- governance KPI trend view produced
- next-phase implementation backlog defined
- executive summary and management action view prepared

### Success indicators
- governance is producing measurable evidence
- recurring issues are being addressed structurally
- managers can see progress and unresolved risks
- next-phase scaling priorities are clear

## Suggested workstreams

| Workstream | Early focus |
|---|---|
| Operating model | Roles, forums, escalation |
| Quality controls | Thresholds, evidence, exception handling |
| Metadata | Critical assets, glossary, dictionary |
| Access governance | Approval pathways, exception logging |
| Issue and change | Intake, remediation, impact review |
| KPI and maturity | Baseline metrics and monitoring |

## Governance rule
The first 90 days should prioritize usable operating discipline over perfect completeness.
