# Contributing

Thank you for your interest in this repository.

This repository is designed as a practical enterprise data governance operating system. Contributions should strengthen clarity, operating discipline, and implementation realism.

## Contribution principles
- Keep content practical, not theoretical
- Tie governance guidance to decisions, controls, workflows, ownership, and measurable outcomes
- Prefer reusable templates, structured models, and implementation-ready artifacts
- Keep naming consistent across folders and files
- When adding a framework, show how it would be used operationally

## Repository conventions
- Use plain, professional language
- Keep markdown scannable with clear headings and tables where useful
- Use consistent terminology for data owner, data steward, control owner, requester, approver, and consumer
- Keep workbook structures aligned with their README tabs
- Do not add placeholder jargon or vague governance language

## Pull request expectations
A good update should answer:
1. What governance problem does this change solve?
2. What operating artifact or control does it improve?
3. What part of the lifecycle does it affect?
4. What would a manager, steward, or analyst do differently because of this change?

## Recommended commit style
- Add governance operating model and structure files
- Add stewardship playbook and RACI workbook
- Add data quality rule library and thresholds
- Add access governance workflow and decision log
- Add implementation roadmap and adoption plan
