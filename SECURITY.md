# Security Policy

## Scope
This repository is a public portfolio repository intended to demonstrate enterprise data governance operating design, templates, workflows, and control structures.

It does not contain production credentials, live confidential business data, or operational access secrets.

## Reporting
If you identify a security issue in repository content, such as:
- accidental exposure of sensitive information
- embedded credentials or tokens
- confidential data in sample artifacts
- unsafe workflow configuration

please report it privately through GitHub or direct contact rather than opening a public issue.

## Handling rules
- Never commit passwords, tokens, keys, or connection strings
- Never upload real personal, financial, operational, or regulated data
- Use synthetic or fully anonymized examples only
- Remove hidden metadata from documents before upload where possible
